# 班级宠物园 🐱

> 游戏化班级管理工具 —— 把积分变成萌宠养成，让班级管理更有趣！

---

## 一、产品定位

### 给谁用
中小学班主任、科任老师、少先队辅导员

### 解决什么问题
传统的班级积分评价制度枯燥、学生积极性不高。班级宠物园将**积分评价**与**电子宠物养成**结合，学生获取的每一分都能用于喂养自己的宠物，宠物会随着积分积累而成长进化，从而让学生从"被动接受评价"变为"主动争取积分"。

### 核心玩法
```
老师给学生加/减分 → 学生积分变化 → 学生用积分喂养宠物 → 宠物获得经验升级 → 解锁新形态
```

### 喂养规则
- 每次喂养消耗 5 积分，随机获得 10-30 经验
- 喂养后扣积分、加经验、自动检测升级
- 每名学生最多领养 5 只宠物
- 不能重复领养同一种宠物

## 六、技术架构

| 层 | 技术 | 说明 |
|---|---|---|
| 前端 | React 18 + Vite + TypeScript + Tailwind CSS | SPA 单页应用 |
| 后端 | Node.js + Express + TypeScript | RESTful API |
| 数据库 | SQLite（sql.js WASM） | 单文件数据库，免安装 |
| 认证 | bcryptjs + JWT | Bearer Token 认证 |
| 验证 | Zod | 请求参数校验 |
| 数据获取 | SWR | 前端数据 fetching + 缓存 |
| 包管理 | npm workspaces | monorepo（shared/server/client） |

### 数据库表（7 张）
- `teachers` — 教师账号
- `classes` — 班级（软删除：is_archived）
- `students` — 学生（软删除：is_active，积分字段：total_points）
- `pets` — 宠物图鉴（13 种种子数据）
- `student_pets` — 领养记录（current_exp、nickname）
- `point_records` — 积分流水（points_change、reason、category）
- `evaluation_rules` — 评价规则（预设 + 自定义）

## 七、本地运行

```bash
# 1. 克隆仓库
git clone https://github.com/manchuyi0319/123.git
cd class-pet-garden

# 2. 安装依赖
npm install

# 3. 启动后端（端口 3001）
npm run dev:server
# 或: cd server && npx tsx src/index.ts

# 4. 启动前端（端口 5173）

# 5. 浏览器打开
```
